# @germondai/blog - Agent Rules

## This is Astro 6 (NOT Next.js, NOT React)

- Components use `.astro` extension
- Pages live in `src/pages/`, content in `src/content/`
- Frontmatter in `.astro` files runs **only at build time** (SSG mode)
- Client JavaScript uses `<script>` tags or `client:*` directives on islands
- `astro:content` provides `getCollection`, `getEntry`, `CollectionEntry`
- `astro check` (not `tsc --noEmit`) for type-checking Astro files
- `output: 'static'` - this is a fully static site, no server runtime

## Runtime: Bun 1.3.14

ALWAYS use `bun` instead of `npm`, `npx`, `yarn`, or `node`:

| Action       | Command                |
| ------------ | ---------------------- |
| Install      | `bun install`          |
| Run script   | `bun run <script>`     |
| Execute tool | `bunx <tool>`          |
| Run TS file  | `bun scripts/file.ts`  |
| Test         | `bun test`             |
| Add package  | `bun add <package>`    |
| Add dev dep  | `bun add -d <package>` |

Never run `npm install`, `npx`, `yarn`, or `node` commands.

## Key conventions

- **Single config file**: users edit ONLY `blog.config.ts` at the project root
- **Theme vars at build time**: CSS vars injected as inline `style` on `<html>` - no runtime JS for theming
- **CSS-first effects**: beams and particles use `@keyframes` in `src/styles/effects.css` - zero JS
- **LensFlare** is the only client island (`client:idle` directive)
- **Biome** for all linting and formatting - no ESLint, no Prettier
- **bun test** for tests - not Jest, not Vitest
- **Path alias**: `@/*` maps to `src/*`
- **Content**: MDX files in `src/content/blog/<lang>/*.mdx`
- **i18n**: default locale has no URL prefix (e.g., `/blog/post`); other locales use prefix (e.g., `/cs/blog/post`)

## Content collection schema

```typescript
// src/content.config.ts  (Astro 6 - uses glob loader, z from 'zod')
import { defineCollection } from 'astro:content'
import { glob } from 'astro/loaders'
import { z } from 'zod'

const blog = defineCollection({
  loader: glob({ pattern: '**/*.{md,mdx}', base: './src/content/blog' }),
  schema: z.object({
    title: z.string(),
    description: z.string(),
    publishDate: z.coerce.date(),
    updatedDate: z.coerce.date().optional(),
    tags: z.array(z.string()).default([]),
    draft: z.boolean().default(false),
    coverImage: z.url().optional(),
  }),
})
```

Note: Astro 6 changed `entry.render()` â†’ `render(entry)` imported from `astro:content`.

## Prebuild scripts (run automatically via `bun run build`)

1. `bun scripts/generate-icons.ts` - downloads avatar â†’ 4 WebP icon sizes in `public/`
2. `bun scripts/generate-font.ts` - downloads font WOFF2 files â†’ `public/fonts/` + writes `src/styles/font.css`
3. `bun scripts/generate-manifest.ts` - derives `public/manifest.json` from config
4. `bun scripts/generate-og.ts` - generates `public/og.webp` (1200Ă—630 WebP)
5. `astro build` - SSG build, output in `dist/`

`bun dev` triggers `predev` â†’ `scripts/setup.ts` (smart: only regenerates what's missing/stale).

Run individually:
- `bun run setup` - regenerate only missing/stale assets
- `bun run icons` - regenerate icons only
- `bun run font` - regenerate font only
- `bun run manifest` - regenerate manifest only
- `bun run og` - regenerate OG image only

## Files never to edit manually

- `src/styles/font.css` - auto-generated by `scripts/generate-font.ts`
- `public/manifest.json` - auto-generated by `scripts/generate-manifest.ts`
- `public/og.webp` - auto-generated by `scripts/generate-og.ts`
